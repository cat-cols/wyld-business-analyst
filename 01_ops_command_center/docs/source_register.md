# Source Register — Project 1

This register documents the primary source extracts used in the Ops Command Center pipeline and shows how they flow into the modeled layers.

| Source Extract / System | Domain | Cadence | Raw Landing Object | Primary Grain | Core Join Keys | Staging / Downstream Models | Known Issues | Status |
|---|---|---|---|---|---|---|---|---|
| `sales_distributor_extract.csv` / distributor | Sales | Daily | `raw.sales_distributor_extract` | Daily sales line (`sale_date + store_code + sku + channel`) | `sale_date`, `store_code`, `sku`, `channel` | `stg.stg_sales_distributor` → `int.int_sales_distributor_dedup` → `mart.fact_sales_distributor_daily` | Messy headers, duplicates, missing `store_code`, channel casing / whitespace drift, mixed date formats | Active |
| `pos_transactions.csv` / POS | Sales | Daily | `raw.pos_transactions_csv` | Transaction line (`txn_id`) | `txn_id`, `txn_date`, `store_code`, `sku` | `stg.stg_pos_transactions` → `int.int_pos_dedup` / `int.int_pos_daily` → `mart.fact_sales_pos_daily` | Bad datetime formats, occasional missing `store_code`, source-system key cleanliness varies | Active |
| `inventory_erp_snapshot.csv` / ERP | Ops | Daily | `raw.inventory_erp_snapshot` | Inventory snapshot (`snapshot_date + store_code + sku`) | `snapshot_date`, `store_code`, `sku` | `stg.stg_inventory_erp` → `int.int_inventory_snapshot_dedup` → `mart.fact_inventory_snapshot_daily` | Negative on-hand exceptions, site code drift (lowercase / hyphen variation) | Active |
| `wms_shipments.csv` / WMS | Ops | Daily | `raw.wms_shipments` | Shipment line (`ship_date + store_code + sku`) | `ship_date`, `store_code`, `sku` | `stg.stg_wms_shipments` → `mart.fact_shipments_daily` | Duplicates, occasional missing SKU | Active |
| `timeclock_punches.csv` / timeclock | People | Daily | `raw.timeclock_punches` | Punch event (`employee_id + punch_ts`) | `employee_id`, `punch_ts`, `store_code` | `stg.stg_timeclock_punches` → `int.int_timeclock_punches_latest` / `int.int_labor_daily_employee` / `int.int_labor_daily` → `mart.fact_labor_daily` | Missing OUT punches, mixed timestamp formats | Active |
| `labor_hours_payroll_export.xlsx` / payroll | People | Weekly | `raw.labor_hours_payroll_export` | Weekly payroll row (`week_ending + store_code + team`) | `week_ending`, `store_code`, `team` | `stg.stg_labor_payroll` → `int.int_labor_daily` / `mart.fact_labor_daily` | Team spelling variation, zero-hour rows with cost | Active |
| `finance_actuals_summary.xlsx` / ERP finance | Finance | Monthly | `raw.finance_actuals_summary` | Monthly KPI category (`month_start + metric_name`) | `month_start`, `metric_name_norm` | `stg.stg_finance_actuals` → `mart.fact_actuals_monthly` → `mart.recon_sales_to_gl_monthly` | Metric-label variance, small source drift vs operational truth | Active |
| GL detail / finance GL | Finance | Monthly | `raw.gl_detail_csv` | GL detail line (account / period detail) | `period_month`, `account`, `metric_category` | currently used as raw finance support; dedicated staging / mart flow is still thin | Present in repo, but not yet a fully first-class modeled source | Partial |
| `account_status.csv` / CRM | Reference | Daily | `raw.account_status` | Account status snapshot (`status_date + store_code`) | `status_date`, `store_code` | `stg.stg_account_status` → `int.int_account_status_current` | Casing / whitespace drift, occasional missing `store_code`, mixed date formats | Active |
| `dispensary_master.csv` / dispensary master | Reference | Daily | `raw.dispensary_master` | Store master snapshot (`as_of_date + store_code`) | `store_code`, `license_number`, `as_of_date` | `stg.stg_dispensary_master` → `int.int_dispensary_latest` / `int.int_dispensary_standardized` → `mart.dim_store` | Messy casing / whitespace, occasional missing keys | Active |
| `sku_distribution_status.csv` / assortment | Reference | Daily | `raw.sku_distribution_status` | Store-SKU status snapshot (`as_of_date + store_code + sku`) | `as_of_date`, `store_code`, `sku` | `stg.stg_sku_distribution_status` → `int.int_sku_distribution_status_dedup` / `int.int_coverage_conformed` → `mart.fact_sku_distribution_status_daily` / `mart.fact_distribution_coverage` | Casing drift, occasional missing `store_code` / `sku`, mixed date formats | Active |

## Notes
- The source-drop manifest in `docs/source_drop_manifest.csv` is the operational log of delivered files and raw loads.
- This register is the business-facing inventory that explains what each source **is**, how often it arrives, and where it lands in the model.
- The cleanest next upgrade is to add **source owner**, **SLA / expected arrival window**, and **criticality tier** once report operations are finalized.
